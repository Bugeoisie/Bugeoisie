Issues in the fuzzing testcacses

Hi libplacebo team,

I found two issues when fuzzing test for libplacebo.

First, in later versions, all of the existing testcases in src/tests/fuzz seems to be importing a deprecated header file ../tests.h. I changed all of them to ../utils.h and now everything compiles well with meson -Dfuzz=true xxxxxx

Second, in src/tests/fuzz/options.c, this fuzzing harness encounters an out-of-bound array access when fuzzer generates input data with length == 0. I believe it's better to have this condition check in __AFL_LOOP().

    while (__AFL_LOOP(100000)) {
        size_t len = __AFL_FUZZ_TESTCASE_LEN;
	if (len < 1) continue;   /* Addition check here */
        buf[len - 1] = '\0'; // ensure proper null termination
        pl_options_load(opts, (const char *) buf);
        //....
